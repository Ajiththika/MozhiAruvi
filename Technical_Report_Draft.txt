1. Abstract / Executive Summary
---------------------------------------------------------
Mozhi Aruvi is a comprehensive e-learning and language platform designed to bridge the gap between language learners and expert tutors. The platform's main objective is to provide a scalable, interactive, and structured educational environment. Built using a modern full-stack web architecture with Next.js (React) on the frontend, Node.js/Express.js on the backend, and MongoDB, the system ensures a fast and reliable user experience. Key outcomes of this project include a robust centralized dashboard for role-based management (Admin, Tutor, Student), dynamic lesson and category management, and multimedia-rich event handling powered by Cloudinary.

---------------------------------------------------------
2. Introduction
---------------------------------------------------------
2.1. Problem statement
Traditional language learning methods often lack structured digital platforms that seamlessly connect students with expert tutors while simultaneously providing engaging lesson materials and community events in a single ecosystem. There is a need for an intuitive, comprehensive platform that handles scheduling, content delivery, and interactive learning smoothly.

2.2. Goals and objectives
- Provide a responsive, user-friendly interface for language learners to access structured lessons.
- Enable administrators to manage lessons, tutors, users, and events efficiently through a unified, secure dashboard.
- Deliver interactive and rich educational content seamlessly by integrating modern media storage solutions.
- Ensure performant and secure role-based authentication to protect user data and platform integrity.

2.3. Scope and limitations
Scope: The system encompasses user authentication, role-based dashboards, a lesson management system, tutor directory, and event creation with image cover support. 
Limitations: The system relies on steady internet connectivity and third-party API services (like Cloudinary) for multimedia rendering. Advanced features like live video streaming are planned for future iterations.

---------------------------------------------------------
3. Requirements
---------------------------------------------------------
Software Requirement:
- Frontend: Next.js, React.js, TypeScript
- Backend: Node.js, Express.js, RESTful APIs
- Database: MongoDB (via Mongoose ODM)
- Authentication: JWT (JSON Web Tokens)
- Styling/UI: Tailwind CSS
- Testing: Manual Testing (Browser based testing, Postman/Thunder Client for API testing)

Hardware Requirement:
- Development: Minimum 8GB RAM, modern multi-core processor.
- Deployment Server: Cloud-based virtual instance (e.g., 1GB RAM, 1vCPU minimum).
- Client side: Any internet-enabled device with a modern web browser.

---------------------------------------------------------
4. Deployment
---------------------------------------------------------
- Frontend Deployment: Typically deployed on platforms like Vercel or Netlify for optimized Next.js delivery.
- Backend Deployment: Hosted on platforms like Render, Heroku, or DigitalOcean Droplets.
- Database: Hosted on MongoDB Atlas for cloud accessibility and scalability.
- File Storage: Cloudinary handles image and multimedia uploads securely.
